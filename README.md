# Data Fusion 2026 — Многоклассовое прогнозирование открытия банковских продуктов

Решение трека ИИ хакатона **Data Fusion 2026** (заказчик трека — банк ВТБ).

## О задаче

Дано: данные о клиентах банка (полностью анонимизированные и обфусцированные — без названий признаков и продуктов, только `cat_feature_*` и `num_feature_*`). Нужно для каждого клиента предсказать вероятность открытия каждого из **41 банковского продукта** — multi-label бинарная классификация, метрика **Macro ROC-AUC** (усреднение ROC-AUC по всем 41 таргетам).

Данные анонимизированы потому, что информация о клиентах банка — персональные данные и банковская тайна, защищённые законом. Из-за этого решение строится исключительно на статистических закономерностях в данных, без какой-либо доменной экспертизы про конкретные банковские продукты.

## Структура репозитория

```
.
├── code1/   — пайплайн предобработки данных и обучения базовых моделей
└── code2/   — финальный ноутбук: многоуровневый стекинг и блендинг
```

> В репозитории папки `code1` и `code2` лежат как `.zip`-архивы — распакуйте их перед запуском.

## code1 — предобработка данных и базовые модели

Набор последовательных Python-скриптов (запускаются по номеру). Каждый шаг сохраняет результат на диск, поэтому пайплайн можно прерывать и продолжать с любого шага.

| Скрипт | Назначение |
|---|---|
| `01_feature_engineering.py` | Загрузка исходных parquet-файлов, очистка и генерация признаков (см. ниже). Сохраняет `features/*.parquet` и `features/meta.json` |
| `02_train_nn.py` | Предобучение Denoising Autoencoder + обучение нейросети (TabM-подобная архитектура с PLR-кодированием, Asymmetric Loss, MixUp, SWA) на 5-fold CV |
| `03_train_lgbm.py` | LightGBM, 41 отдельная бинарная модель на таргет, 5-fold CV |
| `04_train_pyboost.py` | PyBoost (SketchBoost) — multi-output градиентный бустинг на GPU, 5-fold CV |
| `05_train_lgbm_meta.py` | LightGBM с дополнительными кросс-таргетными мета-признаками (OOF-предсказания NN/LGBM/PyBoost по остальным 40 таргетам, без утечки) |
| `06_blend.py` | Ранговый бленд NN + LightGBM + PyBoost с индивидуальными весами на каждый таргет (grid search по OOF) |
| `07_stacking.py` | Мета-обучение второго уровня: Ridge и LightGBM на мета-признаках (предсказания + попарные разности/произведения базовых моделей), финальная комбинация со ранговым блендом |
| `utils.py` | Общие константы (seed, пути, число фолдов) и вспомогательные функции (расчёт Macro ROC-AUC, ранговое преобразование, проверка формата сабмита) |

### Очистка и подготовка данных (`01_feature_engineering.py`)

- **Фильтрация мусора**: из дополнительного набора признаков (`extra_features`, ~2200 числовых колонок) удаляются признаки с долей пропусков выше 99% и константные признаки (нулевая дисперсия).
- **Дедупликация числовых признаков**: по каждой колонке считается MD5-хеш её значений; колонки с одинаковым хешем — точные дубли, лишние удаляются.
- **Удаление дублирующихся категориальных признаков**: заранее определён список дублирующихся `cat_feature_*`-колонок (`DUPLICATE_CATS`), они удаляются из основного набора.
- **Null Pattern PCA**: пропуски в данных не случайны, а идут синхронными группами (общие источники данных). Строится бинарная матрица «пропущено/не пропущено» по всем дополнительным числовым признакам, к ней применяется `TruncatedSVD` — первые 20 компонент используются как отдельные признаки.
- **Индикаторы пропуска по группам**: для нескольких «якорных» столбцов (отдельно в основном и в дополнительном наборе) добавляются групповые бинарные индикаторы пропуска.
- **Индивидуальные индикаторы пропуска**: для каждого числового признака считается корреляция между «значение пропущено» и таргетами; если |corr| > 0.05 хотя бы с одним таргетом — добавляется индикатор `is_null_<feature>`.
- **Частотное кодирование категорий**: `freq_<cat>` — частота встречаемости категории; также частоты для заданных пар взаимодействий категориальных признаков.
- **Числовые diff/ratio-признаки**: заранее отобранные по корреляции с таргетами пары признаков — разности (`diff_*`) и отношения (`ratio_*`).
- **Row-wise статистики**: среднее, стандартное отклонение и асимметрия (skew) по строке клиента — отдельно для основного и для дополнительного наборов признаков.

Итог шага — единый файл с признаками (`features/train_features.parquet`, `features/test_features.parquet`) и описание схемы (`features/meta.json`), которые используют все последующие скрипты.

## code2 — финальная версия: многоуровневый стекинг

`data-fusion.ipynb` — финальный ноутбук (рассчитан на запуск на Kaggle с GPU), на основе которого делался итоговый сабмит. Это более глубокая и развитая версия подхода из `code1`: та же логика предобработки данных, но архитектура ансамбля расширена до полноценного многоуровневого стека.

Архитектура:

1. **Уровень 0 — базовые модели**: нейросеть (та же архитектура, что в `code1/02_train_nn.py`), XGBoost и CatBoost обучаются на GPU прямо в ноутбуке; LightGBM и PyBoost подгружаются из готовых чекпоинтов (для экономии времени на Kaggle).
2. **Честные бленды (L1)**: ранговые бленды базовых моделей с индивидуальными весами на таргет, оценка только по OOF-предсказаниям (без утечки).
3. **Мета-обучение L1–L3**: на каждом уровне обучаются Ridge, XGBoost, CatBoost и PyBoost на предсказаниях предыдущего уровня — каждый уровень даёт новый набор OOF-предсказаний для следующего.
4. **Финальный супербленд**: все лучшие модели с предыдущих уровней объединяются в Sparse Rank Blend с индивидуальным набором весов на каждый из 41 таргетов и финальным переобучением на полных данных.

Все промежуточные модели кэшируются на диск (`.npz` с OOF- и test-предсказаниями), поэтому ноутбук можно перезапускать без повторного обучения уже посчитанных моделей.

### Запуск

Открыть `code2/data-fusion.ipynb` на Kaggle (или локально с GPU), указать пути к исходным parquet-файлам в первой конфигурационной ячейке, выполнить ячейки по порядку.

## Используемые модели

- **Нейросеть**: TabM-подобная архитектура с Piecewise Linear Encoding числовых признаков, Asymmetric Loss (под несбалансированные таргеты), предобученный Denoising Autoencoder для эмбеддингов, MixUp-аугментация и Stochastic Weight Averaging.
- **LightGBM** — 41 отдельная модель (по одной на таргет), параметры подобраны через Optuna.
- **XGBoost, CatBoost** — на GPU.
- **PyBoost (SketchBoost)** — multi-output градиентный бустинг на GPU.
- **Ridge-регрессия** — мета-модель на признаках-предсказаниях, alpha подбирается отдельно под каждый таргет.

## Валидация

Везде используется честная **K-fold кросс-валидация** (5 фолдов, мультилейбл-стратификация через `iterstrat` для бустингов) — итоговые OOF-предсказания собраны так, что модель никогда не предсказывает на объектах, на которых обучалась. Метрика — `roc_auc_score(y_true, y_pred, average="macro")`.
